Claude Code
란?

Anthropic이 만든 AI 기반 CLI 도구.
단순한 챗봇이 아닌, 코드베이스 전체를 이해하는 AI 개발 동료.

$ claude code
일반 챗봇Claude Code
대화만 가능→파일 읽기/편집/생성
맥락 없음→프로젝트 전체 인식
매번 설명→CLAUDE.md 영구 기억
도구 연동 불가→MCP(.mcp.json: TF·Azure·Miro 등)
항상 지시 필요→Hooks 자동 트리거

AI 활용의 단계적 발전

앞쪽은 자료·해석·산출에 가깝고, 뒤로 갈수록 신뢰(검증·하네스)와 재현·운영(자동화·시스템화) 비중이 커진다는 식으로 검색 → 이해 → 생성 → 검증 → 자동화 축에 성숙도를 겹쳐 볼 수 있다. CLI·MCP·Hooks는 특히 후반 단계에서 도구와 흐름을 엮는 역할이 커진다.

3생성코드·문서 산출
4검증하네스로 고정

part1-6

Rules · Skills · Agents · MCP

CLAUDE.md는 저장소 전체 맥락, Rules는 열린 파일·작업에 맞춰 붙는 짧은 규약이다. 아래 표에 Rules부터 MCP까지 한눈에 정리한다.

구분 한 줄 위치 · 예 (cc_edu)
Rules YAML globs가 있으면 패턴과 맞을 때만 주입. mcp.md처럼 globs 없으면 MCP·스킬 흐름 등 전역에 가깝게 적용. .claude/rules/*.md — azure · terraform · mcp · secrets
Skills 사용자가 /이름으로 호출하는 절차 묶음. 스킬 문서가 메인 Claude에게 프롬프트로 주입된다. .claude/skills/ — 예: /tf-plan, /tf-validate, /miro-update
Agents 메인이 필요할 때 띄우는 별도 컨텍스트의 전문 실행체. 역할·도구 범위는 AGENT.md 앞머리에 적는다. .claude/agents/*/AGENT.md — 예: terraform-reviewer, plan-validator, azure-validator
MCP CLI 출력 대신 MCP 도구로 외부 시스템에 접근. 서버는 .mcp.json에 등록하고, settings.json에서 허용한다. 루트 .mcp.json — cc_edu: terraform · azure · miro (stdio). terraform_apply / destroy는 deny.

part2-6

.claude/rules/ — cc_edu 현재 룰 파일

YAML globs가 있으면 해당 패턴의 파일 작업 시에만 주입된다. mcp.md는 globs 없이 MCP·스킬 동작 전역 규칙으로 쓴다.

azure.md
globs: **/*.tf, **/*.tfvars
  • 명명: {약어}-{역할}-{프로젝트}-{환경}-{리전} · Storage 예외 규칙
  • Hub 10.0.0.0/16 · Spoke 10.{n}.0.0/16, PE는 Peering 이후, NSG는 모든 Subnet
  • 태그 필수: environment, owner, project
  • 하드코딩 금지 — 구독·SKU·CIDR 등은 variables.tf + tfvars, 민감값은 sensitive
terraform.md
globs: **/*.tf, **/*.tfvars, **/.terraform.lock.hcl
  • 저장 전 terraform fmt · plan 전 fmt → validate → checkov 순서
  • Checkov 실패 시 skip 자동 추가 금지 — 사용자 결정(human-in-the-loop)
  • plan 후 변경 표 + infracost breakdown --path tfplan (예상 비용만)
  • tags·variables/tfvars 분리 · destroy는 사용자 확인 후
  • 주석 한국어 · provider ~> 4.0 · TF >= 1.5.0
mcp.md
globs 없음 (전역 MCP 규칙)
  • Azure MCP: az login CLI 인증 · npx -y @azure/mcp@latest server start
  • Terraform MCP: npx -y terraform-mcp-server (패키지명 주의)
  • 신규 리소스 전 Registry 검색 · validate·plan/state는 MCP 우선
  • 실청구·비용 API 금지 — infracost(tfplan) 예상만
  • 배포 후 TF state + Azure 실제 리소스 대조 · 아키텍처 변경 시 Miro MCP
  • 스킬은 프롬프트 주입 — tf-apply/destroy는 plan 파일 선행
secrets.md
globs: **/*.tf, **/*.tfvars, **/.env* 등
  • .claudeignore로 tfvars·.env 등 Read 제외 · 답변·PR에 비밀 실값 금지
  • memory·state 인용 시 마스킹 · MCP·비용 정책은 mcp.md와 정합
